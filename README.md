# Paw Bloom 宠物洗护项目设计文档

## 1. 项目概述

Paw Bloom 是一个面向城市宠物主的宠物洗护与造型门店单页官网。项目当前采用纯静态前端实现，通过 `index.html` 组织页面内容，通过 `styles.css` 统一管理视觉样式、布局响应式和交互状态。

网站的核心目标是：

- 快速传达门店定位：安静、干净、有耐心的宠物洗护空间。
- 展示主要服务：基础洗护、猫咪低压护理、精修造型等。
- 说明服务流程：到店评估、分区清洁、耐心吹整、护理反馈。
- 展示价格入口：以透明价格降低用户决策成本。
- 引导预约：通过预约表单收集家长姓名、电话、宠物类型和服务项目。

## 2. 目标用户

主要用户是有宠物洗护、造型或日常护理需求的宠物家长，尤其关注以下体验：

- 宠物在陌生环境中的安全感与舒适度。
- 洗护流程是否透明、专业、可沟通。
- 价格是否清晰，是否能提前了解服务范围。
- 是否方便预约、联系和到店。

## 3. 信息架构

项目使用单页滚动式结构，所有主要信息都在首页完成展示，并通过顶部导航进行锚点跳转。

页面结构如下：

1. 固定顶部导航
   - 品牌标识：Paw Bloom。
   - 导航锚点：服务、流程、价格、预约。
   - 电话入口：`400-816-0921`。

2. 首屏 Hero
   - 使用宠物主题大图作为背景。
   - 展示品牌名称、门店定位和核心服务价值。
   - 提供“立即预约”和“查看价目”两个行动入口。

3. 门店亮点条
   - 1 对 1 护理。
   - 可视化操作间。
   - 敏感皮肤方案。

4. 服务项目
   - 基础沐浴。
   - 猫咪低压护理。
   - 精修造型。

5. 护理流程
   - 入店评估。
   - 分区清洁。
   - 耐心吹整。
   - 护理反馈。

6. 价格模块
   - 小型犬洗护。
   - 全套造型。
   - 猫咪护理。

7. 预约模块
   - 门店营业时间、地址和交通信息。
   - 预约表单。

8. 页脚
   - 品牌名称。
   - 品牌口号。

## 4. 功能设计

### 4.1 导航与跳转

顶部导航使用固定定位，用户在页面任意位置都可以快速跳转到服务、流程、价格和预约模块。页面开启 `scroll-behavior: smooth`，锚点跳转具备平滑滚动体验。

### 4.2 服务展示

服务模块采用横向轮播布局，每张卡片包含：

- 服务图片。
- 服务名称。
- 简短服务说明。

轮播支持左右切换、圆点定位和移动端横滑，卡片使用统一边框、阴影、圆角和图片比例。

### 4.3 护理流程

护理流程采用编号步骤卡片，强调服务的专业顺序。每一步包含编号、标题和说明，帮助用户理解到店后的实际体验。

### 4.4 价格展示

价格模块使用三张价格卡片，其中“全套造型”为重点卡片，使用深绿色背景增强视觉权重。每张价格卡片包含：

- 服务名称。
- 起始价格。
- 适用范围说明。
- 预约入口。

### 4.5 预约表单

预约模块包含一个静态表单，字段包括：

- 家长姓名。
- 联系电话。
- 宠物类型。
- 想预约的服务。

当前表单按钮类型为 `button`，尚未接入提交逻辑。后续可接入后端接口、表单服务、短信提醒或门店 CRM。

## 5. 视觉设计

### 5.1 设计关键词

- 温和。
- 专业。
- 干净。
- 可信赖。
- 有呼吸感。

整体视觉避免过度活泼的宠物店风格，转向更安静、护理感更强的门店品牌形象。

### 5.2 色彩系统

核心颜色定义在 `:root` 中：

- `--ink: #18211f`：主要文字与深色页脚。
- `--muted: #5d6864`：辅助说明文字。
- `--line: #dce4df`：边框与分隔线。
- `--paper: #f7f2ea`：页面底色。
- `--surface: #ffffff`：卡片和内容表面。
- `--sage: #7aa28f`：鼠尾草绿，传达自然护理感。
- `--sage-dark: #315f50`：深绿色，用于强调卡片和品牌色。
- `--coral: #ef806f`：珊瑚色，用于按钮、强调文字和价格。
- `--gold: #dba94b`：预留暖色强调色。
- `--blue: #d9ebf3`：图片占位和清爽辅助色。

### 5.3 字体与排版

字体栈：

```css
"Microsoft YaHei", "PingFang SC", "Segoe UI", sans-serif
```

排版策略：

- 首屏标题使用大字号，强化品牌记忆。
- 内容区标题使用响应式字号，适配桌面和移动端。
- 正文保持 `line-height: 1.6`，提升中文阅读舒适度。
- 字距统一为 `0`，避免中文排版出现不自然压缩。

### 5.4 图片策略

项目混合使用远程图片与项目内 AI 生成图片作为视觉素材：

- 首屏背景图用于建立宠物护理场景。
- 常用洗护项目的服务卡片图保存在 `assets/services/`，分别对应基础沐浴、猫咪低压护理和精修造型。

后续上线前可继续替换为门店真实图片，增强信任感和品牌识别度。

## 6. 布局与响应式设计

项目使用 CSS Grid 和 Flexbox 完成主要布局。

桌面端：

- 顶部导航为三栏结构：品牌、导航、电话。
- 服务模块使用单张图文轮播，价格模块使用三列网格。
- 护理流程为左右双栏，右侧步骤为两列。
- 预约模块为左右双栏，左侧文案，右侧表单。

平板端与窄屏：

- 当视口小于 `900px` 时，导航隐藏，主要多列布局切换为单列。
- 护理步骤在中等屏幕下保留两列，保持信息密度。

移动端：

- 当视口小于 `640px` 时，品牌文字隐藏，仅保留品牌标识。
- 按钮宽度变为 100%，方便触控。
- 服务卡片图片高度降低，减少首屏外滚动压力。
- 预约表单切换为单列，避免输入控件拥挤。
- 页脚改为纵向排列。

## 7. 技术架构

当前项目是零构建静态站点，文件职责如下：

```text
pet_care/
├── assets/       # 项目内图片资源
├── index.html    # 页面结构、文案、图片和表单内容
├── styles.css    # 视觉样式、布局、响应式规则
├── README.md     # 项目说明与设计文档
└── .gitignore    # Git 忽略配置
```

技术特点：

- 无前端框架依赖。
- 无构建工具依赖。
- 可直接通过浏览器打开 `index.html` 预览。
- 样式集中在单个 CSS 文件中，便于小型项目维护。
- 使用语义化标签，如 `header`、`main`、`section`、`article`、`footer`、`address`。

## 8. 可访问性设计

当前页面已经包含部分可访问性基础：

- 页面语言设置为 `zh-CN`。
- 使用语义化 HTML 结构。
- 主要模块通过 `aria-labelledby` 关联标题。
- 导航和快捷操作使用 `aria-label`。
- 图片提供 `alt` 文本。
- 电话使用 `tel:` 链接，方便移动设备直接拨打。

后续可继续优化：

- 为预约表单增加必填校验和错误提示。
- 提升按钮、链接和输入框的键盘焦点样式。
- 确保所有图片 `alt` 文本在编码正确后可准确表达内容。
- 为表单提交结果增加屏幕阅读器可感知的反馈区域。

## 9. 当前限制

当前版本仍是展示型静态页面，存在以下限制：

- 预约表单没有真实提交能力。
- 首屏背景仍依赖外部 Unsplash 链接，离线或网络异常时会影响展示。
- 页面文案在当前 `index.html` 中存在编码显示异常，需要统一为 UTF-8 后修正。
- 未配置 SEO 结构化数据。
- 未配置自动化测试、代码格式化或部署脚本。

## 10. 后续迭代建议

优先级较高的迭代：

- 修复 `index.html` 中文编码异常，统一保存为 UTF-8。
- 接入预约提交逻辑，可以先使用静态表单服务，再扩展为后端接口。
- 使用门店真实照片替换远程素材。
- 增加表单校验、提交成功状态和失败提示。
- 补充地图入口、营业状态和节假日说明。

中长期迭代：

- 增加服务详情页或弹窗，展示护理包含项、适用宠物和注意事项。
- 增加会员卡、套餐和优惠活动模块。
- 增加用户评价和护理前后对比图。
- 增加基础 SEO：Open Graph、结构化数据、站点图标。
- 引入轻量 JavaScript 管理表单交互和移动端导航。
- 部署到静态托管平台，如 GitHub Pages、Netlify、Vercel 或对象存储 CDN。

## 11. 维护规范

建议后续维护时遵循以下约定：

- 页面结构修改优先在 `index.html` 中完成。
- 视觉和响应式调整统一写入 `styles.css`。
- 新增颜色、阴影、间距等全局视觉变量优先放入 `:root`。
- 新增模块时保持当前页面的圆角、间距和卡片风格一致。
- 上线前检查桌面端、平板端和移动端三个断点。
- 修改中文文案时确保文件以 UTF-8 编码保存。
